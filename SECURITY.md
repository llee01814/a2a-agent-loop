# Security Policy

Please report security issues privately through GitHub security advisories when enabled, or by opening an issue that avoids publishing exploit details.

## Scope

- command adapter sandboxing
- dashboard control endpoints
- prompt-injection resistance
- accidental secret exposure
- unsafe package install recommendations

## Out of scope

- bypassing AI provider safety systems
- credential theft
- malware or persistence techniques

## Safe defaults

The demo uses mock agents and requires no secrets.
